# Phase 1 - P0 Correctness Fixes

## 1. Fix price sanitisation for comma-formatted prices

### Problem

A price such as `20,250.25` appears to normalise to `20.25025`. This is wrong for futures/index prices. It should become `20250.25`.

### Files

- `assets/app.js`
- `tests/smoke.js`

### Steps

1. Find the price/numeric cleaning helper in `assets/app.js`.
2. Update the logic so thousands commas are removed.
3. Preserve decimal points.
4. Reject or safely handle non-numeric values such as `N/A`.
5. Update smoke test fixtures:
   - `20,250.25` must normalise to `20250.25`.
   - `20,000` must normalise to `20000`.
   - `20250.25` must remain `20250.25`.
   - `N/A` must not break distance calculations.
6. Run `node tests/smoke.js`.

### Acceptance Criteria

- DOL distance is mathematically correct.
- Price Map points and percentage are correct.
- R:R calculations are not distorted by commas.

## 3. Strengthen import validation

### Problem

Import currently normalises broad incoming payloads. It needs clearer validation and safer user feedback.

### Files

- `assets/app.js`
- `tests/smoke.js`
- `README.md`

### Steps

1. Review `extractPayload()` and `importCards()`.
2. Accept known shapes only:
   - current schema export
   - recognised legacy export shapes
   - direct array only if used for legacy migration/testing
3. Reject unknown object payloads.
4. Never wipe existing saved cards on invalid import.
5. Return a useful result object:
   - imported count
   - skipped count
   - duplicate count if practical
   - error/message if rejected
6. Add smoke tests for:
   - valid current export import
   - legacy import
   - duplicate id import
   - older duplicate does not overwrite newer local card
   - invalid object import does not change saved cards
   - non-JSON string import does not change saved cards
7. Run `node tests/smoke.js`.

### Acceptance Criteria

- Bad imports are rejected safely.
- Valid imports preserve all fields.
- Existing cards are never wiped by failed imports.
