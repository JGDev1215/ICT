# Daily Reports

> Status: Current
> Last reviewed: 2026-07-09
> Source of truth: Yes

This folder stores end-of-session reports generated by an LLM or agent. Use it as a historical log of what changed, what was tested, what issues were found, and what should happen next.

## When To Create A Report

Create a daily report when the user asks for an end-of-session summary, daily report, session report, or historical log.

## Naming Convention

Use this filename pattern:

```text
YYYY-MM-DD-session-report.md
```

If multiple reports are needed on the same day, append a short suffix:

```text
YYYY-MM-DD-session-report-2.md
YYYY-MM-DD-supabase-session-report.md
```

## Report Template

```md
# Daily Report - YYYY-MM-DD

> Status: Historical
> Last reviewed: YYYY-MM-DD
> Source of truth: No

## Session Summary

## Actions Taken

## Files Changed

## Issues Faced

## Tests And Checks

## Decisions Made

## Current Git State

## Risks Or Open Questions

## Recommended Next Steps
```

## Agent Notes

- Keep reports factual and concise.
- Link to commits, files, and docs where useful.
- Do not treat daily reports as current requirements; they are historical records.
- If a daily report creates new action items, put the implementation plan in `docs/plans/` when work begins.
