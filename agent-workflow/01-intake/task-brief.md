# Task Brief

## Original User Task

Implement the proposed plan to replace the sloppy Supabase Profile panel with single-user admin access.

## Objective

Make the Profile account/backup area user-friendly for a single-user app while preserving Supabase-backed Focus Card sync behind the scenes.

## Repo Findings

- App is static HTML/CSS/plain JS.
- Profile server-sync UI is generated by `supabasePanelHtml()` in `assets/app.js`.
- Supabase auth internals already exist and should remain.
- Runtime config is initialized in `assets/config.js`.
- Current visible version is `v0.8.1`.
- Cache-busted asset strings are in `index.html` and `service-worker.js`.
- Smoke tests assert Supabase signup UI today and must be updated.
- Worktree has unrelated local changes in `AGENTS.md` and untracked prompt/report files; do not touch them.

## Assumptions

- Username is exactly `admin`.
- UI password is entered by the user and passed to Supabase Auth; no hardcoded browser password check beyond username validation.
- Backing Supabase Auth email defaults to `admin@ict.local`.
- `window.ICT_CONFIG.adminSupabaseEmail` can override the backing email.
- Static `admin/admin` is a convenience gate, not strong security.

## Out of Scope

- Replacing Supabase storage.
- Building multi-user auth.
- Adding server-side security.
- Redesigning unrelated Profile settings/data tools.
- Committing or pushing unless separately requested.

## Success Criteria

- [ ] Profile panel says `Account & Backup`, not `Supabase Focus Cards`.
- [ ] Normal logged-out UI has username/password and `Sign in`.
- [ ] Normal UI does not expose project URL, signup, retry sync, or Supabase email confirmation copy.
- [ ] `admin` maps to `admin@ict.local` by default.
- [ ] First-sync choice copy is user-friendly.
- [ ] Supabase user `admin@ict.local` exists and is confirmed.
- [ ] Version/cache strings are bumped.
- [ ] README/CHANGELOG/tests are updated.
- [ ] `node tests/smoke.js` passes.
